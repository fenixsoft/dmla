# 自动化调参

选择超参数是机器学习开发中最耗时的环节之一。学习率设高了训练不收敛，设低了收敛太慢；正则化强度太小模型过拟合，太大又欠拟合；网络层数、批量大小、Dropout 率……这些超参数的组合空间是指数级的，靠人工试错无异于大海捞针。自动化调参（Hyperparameter Optimization，HPO）将这个搜索过程系统化：用更少的试验次数，找到更优的参数组合。

## 自动化调参的核心问题

### 超参数搜索的挑战

- 搜索空间的高维性：一个典型的深度学习模型有 10+ 个超参数，每个有连续或离散的取值范围
- 评估成本的高昂性：每次评估需要完整训练一个模型，耗时从分钟到天不等
- 目标函数的黑箱性：超参数与模型性能之间的关系没有解析表达式，无法求梯度
- 评估结果的噪声性：相同超参数在不同随机种子下可能产生不同的结果
- 多目标优化：精度、延迟、模型大小等多个目标往往互相冲突

### 手动调参 vs 自动化调参

- 手动调参依赖经验直觉：研究者根据过往经验设定参数，容易陷入局部最优
- 网格搜索（Grid Search）的暴力枚举：搜索空间随参数数量指数增长，不可持续
- 随机搜索（Random Search）的启示：伯格斯特拉（Bergstra）和本吉奥（Bengio）2012 年的论文证明，随机搜索在同等预算下通常优于网格搜索
- 自动化调参的目标：用最少的试验次数，找到接近最优的超参数组合

### 搜索空间的设计

- 连续参数：学习率（通常在对数空间搜索，如 [1e-5, 1e-1]）、Dropout 率（[0.0, 0.5]）
- 离散参数：批量大小（[16, 32, 64, 128]）、优化器类型（SGD, Adam, AdamW）
- 条件参数：选择 Adam 优化器时才有 beta1/beta2 参数，选择 SGD 时才有 momentum 参数
- 搜索空间的先验：基于领域知识缩小搜索范围，如学习率通常在 [1e-5, 1e-1] 而非 [0, 1000]
- 搜索空间的定义方式：统一接口描述参数名、类型、范围、先验分布

## 基础搜索策略

### 网格搜索

- 预定义每个参数的候选值集合，穷举所有组合
- 优势：简单、可并行、覆盖均匀
- 劣势：维度灾难——10 个参数各 5 个取值 = 5^10 ≈ 1000 万次试验
- 适用场景：参数少（<=4）且候选值有限时
- 网格搜索的浪费：当一个参数对结果影响不大时，该参数的所有取值都被无效遍历

### 随机搜索

- 每次从搜索空间中随机采样一组参数
- 伯格斯特拉和本吉奥的关键洞察：当某些参数对结果影响大于其他参数时，随机搜索更容易发现好的组合
- 随机搜索效率更高的数学解释：假设只有 1/5 的参数真正重要，网格搜索 5^10 次试验只能为重要参数各尝试 5^2 个值，随机搜索 100 次试验就能为重要参数尝试 100 个不同的值
- 适用场景：参数多、预算有限时作为基线方法
- 随机搜索的局限：不考虑历史试验结果，无法利用已积累的信息

### 半随机策略：Halving 与 Successive Halving

- 连续减半（Successive Halving, SH）：先以少量资源（如 1 个 epoch）评估 N 组参数，淘汰最差的一半，对剩余参数加倍资源继续评估，反复迭代
- SH 的超参数：初始参数数量 N 和淘汰比例，需要预先设定总预算
- SH 的资源分配逻辑：将更多资源分配给有潜力的参数组合，避免在劣质组合上浪费完整训练预算
- SH 的问题：N 的选择对结果影响大——N 太大导致初始资源太少，好参数可能因噪声被误淘汰；N 太小则覆盖不足

## 贝叶斯优化

### 贝叶斯优化的基本框架

- 贝叶斯优化将超参数搜索建模为序列决策问题：每次选择最有可能改进当前最优的参数组合
- 两个核心组件：代理模型（Surrogate Model）近似目标函数，采集函数（Acquisition Function）决定下一个评估点
- 迭代过程：历史数据 -> 更新代理模型 -> 最大化采集函数 -> 评估新参数 -> 更新历史数据
- 与随机搜索的区别：贝叶斯优化利用历史信息指导搜索方向，而非盲目采样

### 代理模型

- 高斯过程（Gaussian Process, GP）：贝叶斯优化最经典的代理模型，对目标函数提供均值和不确定性估计
- GP 的优势：在连续空间上提供平滑的近似，天然输出不确定性
- GP 的局限：时间复杂度 O(n^3)（n 为历史观测数），不适合大量观测；难以处理高维空间和分类参数
- 替代代理模型：Tree-structured Parzen Estimator（TPE）、随机森林（SMAC）、神经网络（BNN, Deep Ensemble）
- TPE 的工作原理：将参数空间分为"好"（loss < 阈值）和"差"（loss >= 阈值）两组，分别建模密度函数 l(x) 和 g(x)，选择 l(x)/g(x) 最大的点作为下一个评估点

### 采集函数

- Probability of Improvement (PI)：选择最有可能超过当前最优值的点
- Expected Improvement (EI)：选择期望改进量最大的点，同时考虑改进概率和改进幅度
- Upper Confidence Bound (UCB)：选择均值 + k 倍标准差最大的点，k 控制探索-利用的平衡
- 采集函数的探索-利用权衡：探索（高不确定性区域，可能有意想不到的好结果）vs 利用（均值高的区域，大概率有改进）
- 采集函数的优化：采集函数通常非凸，需要用梯度下降或随机采样等方法求解

### 贝叶斯优化的工程挑战

- 并行化：标准贝叶斯优化是串行的，每次只建议一个评估点。并行策略包括常数欺骗（Constant Liar）、批量采集函数优化
- 多保真度优化：利用低保真度评估（少量 epoch、数据子集、低分辨率）快速筛选参数，高保真度评估只在有潜力的参数上执行
- 多目标优化：同时优化精度和延迟，寻找帕累托前沿
- 约束优化：参数必须满足某些约束（如模型大小 < 100MB），违反约束的参数不评估

## 高级搜索策略

### 多保真度方法

- Hyperband：结合 Successive Halving 和随机搜索，自动分配不同保真度级别的资源
- Hyperband 的双层循环：外层循环尝试不同的初始参数数量 N，内层执行 Successive Halving
- Hyperband 的优势：无需手动设定 N，自动平衡探索（大 N 少资源）和利用（小 N 多资源）
- BOHB：Hyperband + 贝叶斯优化，在 Hyperband 的每一轮用 TPE 替代随机采样，兼顾效率和利用历史信息
- ASHA（Asynchronous Successive Halving）：异步版本的 SH，支持分布式并行执行，无需等待同一轮的所有试验完成

### 群体智能与进化方法

- 遗传算法：将超参数编码为"基因"，通过选择、交叉、变异操作迭代优化种群
- 进化策略（Evolution Strategy）：不使用交叉，仅通过变异和选择优化，适合连续参数空间
- 种群大小与训练代数的平衡：大种群提供更好的多样性，但每代需要更多评估
- 适用场景：搜索空间包含条件参数或非凸区域时，进化方法可能比贝叶斯优化更鲁棒

### 元学习与迁移学习

- 元学习（Meta-Learning）：利用历史调参任务的经验，为新任务推荐初始参数或搜索空间
- 学习曲线外推：根据早期训练阶段的指标趋势，预测最终性能，提前终止劣质试验
- 迁移学习：将相似任务上的调参结果作为先验，加速新任务的搜索
- 基准数据集的作用：OpenML、HPO-Bench 等公开调参基准为元学习提供训练数据

## 自动化调参的工程实践

### 调参与实验追踪的集成

- 每次调参试验应作为一次实验记录到实验追踪系统中，包含超参数、指标和模型产物
- 调参框架（Optuna、Ray Tune）通常提供与实验追踪系统的集成接口
- 调参结果的汇总：搜索空间、最优参数、参数重要性排序、搜索轨迹
- 调参的可复现性：记录随机种子、搜索策略配置和框架版本

### 资源分配与调度

- 调参任务的资源预算：固定总 GPU 小时数，或固定试验次数
- 早停策略（Early Stopping）：检测到试验不太可能超过当前最优时提前终止，释放资源
- 资源竞争：多个调参任务共享 GPU 集群时，需要公平调度和优先级管理
- 检查点恢复：调参过程中断后，应能从上次进度恢复，避免重复评估

### 搜索结果分析

- 参数重要性排序：哪些超参数对性能影响最大，有助于缩小后续搜索范围
- 参数交互分析：某些参数组合产生协同效应（如学习率和批量大小），需要联合搜索
- 搜索轨迹可视化：展示目标函数值随试验次数的变化，判断搜索是否收敛
- 搜索空间覆盖率：评估搜索是否充分探索了参数空间，还是集中在某个局部区域
