# 漂移检测

机器学习模型建立在"训练数据的分布与未来数据的分布相同"这一假设之上。但现实世界中，数据分布无时无刻不在变化：季节更替改变消费行为，政策调整影响市场走势，突发事件颠覆历史模式。这种数据分布随时间变化的现象称为数据漂移（Data Drift），它是模型性能退化的首要原因。漂移检测要回答的问题是：分布变了吗？变了多少？是哪种类型的漂移？是否需要采取行动？

## 漂移检测的核心问题

### 漂移的必然性

- 真实世界的数据生成过程是非平稳的（Non-stationary），分布漂移是常态而非例外
- 漂移的时间尺度：缓慢渐变（消费者偏好逐年变化）、周期性波动（季节性购买模式）、突发剧变（疫情导致在线行为剧变）
- 静态模型的悖论：模型在训练数据上表现越好，对分布变化的适应力可能越差（过拟合于特定分布）
- 漂移不是异常，漂移是数据在正常演化；异常是数据采集或处理错误，漂移是真实世界的反映

### 漂移与模型退化的因果关系

- 数据漂移不必然导致模型退化：某些维度的漂移可能不影响模型决策边界
- 模型退化不必然由数据漂移导致：可能是特征管道故障、标签定义变更等其他原因
- 漂移检测与模型性能监控的关系：漂移检测是性能退化的早期预警信号，但不是等价信号
- 从漂移到退化的传导链：数据漂移 -> 特征分布变化 -> 模型输入偏移 -> 预测偏差累积 -> 性能退化

### 漂移检测的粒度

- 输入漂移（Covariate Shift）：输入特征 X 的分布变化，P(X) != P_ref(X)
- 概念漂移（Concept Drift）：给定输入时输出标签的条件概率变化，P(Y|X) != P_ref(Y|X)
- 标签漂移（Label Shift）：输出标签的边际分布变化，P(Y) != P_ref(Y)
- 三种漂移类型的区别：输入漂移是"问题的样子变了"，概念漂移是"问题的答案变了"，标签漂移是"答案的比例变了"
- 不同漂移类型对应不同的应对策略

## 漂移类型的深入分析

### 输入漂移

- 输入漂移是最容易检测的类型：不需要标签，只需比较特征分布
- 渐变型输入漂移：特征分布缓慢偏移，如用户年龄分布随产品定位调整而变化
- 突发型输入漂移：特征分布突然跳变，如数据采集传感器故障、特征管道变更
- 输入漂移的影响评估：并非所有输入漂移都影响模型性能，只有影响决策边界附近区域的漂移才需要关注
- 虚荣漂移（Vanity Drift）：检测到的漂移在统计上显著但业务上无关紧要，频繁误报的常见原因

### 概念漂移

- 概念漂移是最危险的类型：输入分布不变，但输入与输出的关系改变了
- 渐变型概念漂移：用户偏好的缓慢转变，如对某类商品的兴趣逐渐降低
- 突发型概念漂移：政策变化导致的风险规则重写，如疫情期间违约定义的改变
- 循环型概念漂移：季节性模式，如冬季和夏季的用电量预测模型需要不同的决策逻辑
- 概念漂移的检测困难：需要标签才能确认 P(Y|X) 是否变化，而标签往往延迟

### 标签漂移

- 标签漂移：正负样本的比例发生变化，如欺诈率从 1% 上升到 3%
- 标签漂移与输入漂移的耦合：标签漂移往往伴随输入漂移，但二者可以独立发生
- 标签漂移对模型的影响：模型输出的预测概率不再准确，校准关系被打破
- 标签漂移的检测：可以直接统计正负样本比例变化，也可通过模型预测分布间接推断

## 统计检测方法

### 单变量漂移检测

- Kolmogorov-Smirnov (KS) 检验：比较两个经验分布函数的最大差异，适用于连续型特征
- Population Stability Index (PSI)：对两个分布分箱后计算相对熵的加权和，金融行业广泛使用
- 卡方检验（Chi-Square Test）：比较分类特征的频率分布，适用于离散型特征
- Wasserstein 距离：衡量将一个分布"搬运"为另一个分布的最小代价，对分布平移敏感
- 单变量检测的局限：只能逐个特征检测，无法捕捉特征间的联合分布变化

### 多变量漂移检测

- 多变量漂移的挑战：特征间的相关性变化可能不被任何单变量检测捕捉
- 基于降维的方法：先用 PCA/Autoencoder 将高维特征降到低维，再在低维空间做分布检测
- 基于分类器的方法：训练一个二分类器区分参考数据和新数据，分类器的准确率越高，漂移越严重
- Maximum Mean Discrepancy (MMD)：在再生核希尔伯特空间（RKHS）中比较两个分布的均值差异
- 多变量方法的权衡：检测能力更强但计算成本更高，且结果更难解释

### 时序漂移检测

- ADWIN（ADaptive WINdowing）：自适应调整窗口大小，当两个子窗口的均值差异超过阈值时报告漂移
- Page-Hinkley 检验：检测序列均值的变化，对渐进漂移敏感
- CUSUM（Cumulative Sum）：累积偏差超过阈值时报告漂移，工业界经典方法
- 时序检测的窗口选择：窗口太大检测延迟高，窗口太小噪声影响大
- 在线检测 vs 离线检测：在线检测逐条处理，适合流式数据；离线检测批量处理，适合周期性检查

## 漂移检测的工程实践

### 参考分布的选择

- 参考分布（Reference Distribution）是漂移检测的基准，通常使用训练数据或最近一次重训练的数据
- 固定参考 vs 滚动参考：固定参考能检测累积漂移但可能误报，滚动参考能适应渐变但可能漏报
- 参考分布的更新时机：模型重训练后更新参考分布，与模型版本绑定
- 参考分布的存储：存储原始数据不可行（量大），通常存储统计摘要（直方图分位数、分箱频率）

### 检测频率与延迟

- 检测频率的选择：实时（每条请求）、分钟级、小时级、天级，频率越高成本越高
- 检测延迟与行动延迟的权衡：检测延迟低但行动延迟高（检测到了但重训练需要时间）
- 累积窗口检测：在滑动窗口内累积统计量，减少单次采样的噪声影响
- 检测结果的时间粒度：检测结果应标注时间窗口和样本量，避免误解读

### 漂移严重程度评估

- 统计显著性 vs 业务显著性：统计上显著的漂移未必需要采取行动
- 漂移影响的量化：评估漂移对模型性能的实际影响，将漂移检测与性能监控关联
- 影响评分：综合考虑漂移幅度、影响范围、模型敏感度，给出 0-1 的影响评分
- 行动阈值：根据影响评分决定是否触发重训练，避免不必要的模型更新

### 漂移检测与自动化响应

- 检测到漂移后的自动化响应链：漂移告警 -> 根因分析 -> 决策（重训练/调整/忽略）-> 执行 -> 验证
- 自动重训练的触发条件：漂移影响评分超过阈值且持续 N 个检测窗口
- 自动重训练的风险：新数据可能包含异常值或标注错误，自动训练可能产生更差的模型
- 人工审核环节：自动重训练后的模型应经过验证才能部署，不应完全跳过审批流程
