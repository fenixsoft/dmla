## 量化与精度优化

在前面的章节中，我们讨论了如何更高效地利用 GPU 的显存和算力。但还有一条更根本的优化路径：减少资源需求本身。量化通过降低模型权重和 KV Cache 的数值精度，直接减少显存占用、计算量和带宽需求，是降低推理成本最直接的手段。

### 量化的基本原理

量化的核心思想是用更低精度的数值格式表示模型权重和/或激活值。从 float32 到 float16 已经是一种 2 倍压缩，但推理优化的量化目标更激进：int8（4 倍压缩）甚至 int4（8 倍压缩）。精度越低，每个数值占用的存储空间越小，GPU 的 Tensor Core 在低精度下的吞吐量也越高——int8 的 Tensor Core 吞吐量是 float16 的 2 倍，int4 的吞吐量更高。

量化分为两类。**训练后量化**（Post-Training Quantization，PTQ）不需要重新训练模型，直接对已有模型的权重进行量化，实现简单但精度损失可能较大。**量化感知训练**（Quantization-Aware Training，QAT）在训练过程中模拟量化误差，让模型学会在低精度下保持性能，精度损失更小但需要额外的训练成本。对于 LLM 推理，PTQ 是主流选择，因为大模型的训练成本极高，而 PTQ 在 4-bit 量化下仍能保留 97% 以上的性能，性价比远高于 QAT。

量化对推理的收益体现在三个方面：显存占用减少，更多请求可以同时处理；计算速度提升，低精度 Tensor Core 的吞吐量更高；带宽需求降低，每字节传输更多参数。这三项收益叠加，使得量化成为推理优化中投入产出比最高的技术之一。

### 权重量化：从 GPTQ 到 AWQ

权重量化是量化技术中研究最深入的方向。模型权重是静态的，量化后不再变化，因此可以在量化时进行精细的误差补偿。

2022 年，奥地利格拉茨技术大学的研究员弗兰塔·伊莱亚斯（Elias Frantar）等人在论文《GPTQ: Accurate Post-Training Quantization for Generative Pre-trained Transformers》中提出了 GPTQ 方法。GPTQ 基于近似二阶信息，逐层量化权重矩阵，通过 Hessian 矩阵补偿量化误差。具体来说，当某个权重量化后产生误差时，GPTQ 利用 Hessian 矩阵的信息调整相邻权重的值，使得整体误差最小化。这种方法在 4-bit 量化下仍能保留原始模型 99% 的性能，但缺点是量化过程较慢 —— 需要逐行计算 Hessian 矩阵，一个 70B 模型的量化可能需要数小时。

2023 年，麻省理工学院（MIT）的林智杰（Ji Lin）等人在论文《AWQ: Activation-aware Weight Quantization for LLM Compression and Acceleration》中提出了 AWQ 方法，从另一个角度解决了权重量化问题。AWQ 的关键发现是：权重中只有约 1% 的通道对激活值敏感，这些"重要通道"的量化误差对模型性能影响极大，而其余 99% 的通道即使激进量化也不会显著影响性能。AWQ 对重要通道保持较高精度（通过缩放因子放大后再量化），其余通道激进量化。这种方法比 GPTQ 更快（无需逐行计算 Hessian），且精度相当，已成为生产环境中最常用的权重量化方案之一。

同年，保加利亚开发者乔治·格尔加诺夫（Georgi Gerganov）开发了 llama.cpp 项目，并提出了 GGUF 量化格式。GGUF 为 CPU 推理优化，支持多种量化精度（2-bit 到 8-bit 混合），使得 70B 模型可以在消费级硬件上运行。GGUF 的核心创新是按重要性对不同层使用不同精度——Attention 层使用较高精度，MLP 层使用较低精度，在保持性能的同时最大化压缩率。

### KV Cache 量化

KV Cache 量化与权重量化面对的挑战不同。权重是静态的，量化后不再变化，可以在量化时进行精细的误差分析和补偿。KV Cache 则是动态的，每个请求的 KV Cache 都不同，且随生成过程不断增长，无法提前分析其数值分布。

KV Cache 量化的主要挑战在于量化精度对注意力计算的影响。注意力分数的计算公式为 $\text{softmax}(QK^T / \sqrt{d})$，其中 $QK^T$ 是 Query 和 Key 的点积。Key 的量化误差会直接影响点积的结果，进而改变 Softmax 后的注意力分布。如果量化导致注意力分布偏移，模型可能关注错误的 token，生成质量就会下降。

实践中，FP8 E5M2 格式是当前最主流的 KV Cache 量化方案。FP8 有两种格式：E4M3（4 位指数、3 位尾数，精度更高但范围更小）和 E5M2（5 位指数、2 位尾数，精度更低但范围更大）。KV Cache 量化使用 E5M2 而非 E4M3，原因是 KV Cache 中的数值范围更大。Key 和 Value 向量是 Transformer 各层的输出，经过 LayerNorm 归一化后大部分数值集中在较小范围内，但少数异常值（Outlier）可能非常大。这些异常值在注意力计算中起关键作用，如果被截断会导致严重的精度损失。E5M2 格式可表示的数值范围更大（最大约 57344，而 E4M3 最大约 448），能容纳异常值而不溢出。相比之下，权重是静态的，在量化时可以逐通道分析数值范围并选择合适的缩放因子，因此权重 FP8 量化通常使用 E4M3（精度更高，范围够用）。

vLLM 的 KV Cache FP8 量化是当前最成熟的实现。在 A100 上，FP8 量化将 KV Cache 显存减半，吞吐量提升 30-50%，而生成质量的影响在大多数任务上可忽略。这使得 KV Cache 量化成为推理服务中几乎"免费"的优化 —— 只需修改配置即可启用，几乎没有副作用。

### 量化的精度 - 效率权衡

量化不是免费的午餐。精度越低，显存和速度收益越大，但生成质量损失也越大。4-bit 量化在简单任务（如文本分类、情感分析）上损失可忽略，但在数学推理和代码生成等需要精确计算的任务上可能有明显退化。这是因为低精度量化引入的误差会累积 —— 一个 70B 模型有 80 层，每层的量化误差都会传递到下一层，经过 80 层的累积后，即使每层的误差很小，最终的影响也可能不可忽视。

**混合精度量化**是缓解精度损失的有效策略，对模型的不同层使用不同的量化精度。研究表明，Attention 层（特别是 QK 点积计算）的精度对生成质量影响更大，可以保持 float16 或 int8 精度；MLP 层的精度影响较小，可以更激进地量化到 int4。这种"好钢用在刀刃上"的策略，在保持整体性能的同时最大化压缩率。

量化效果的评估需要多维度进行。Perplexity（困惑度）评估量化对语言建模能力的影响，是最基本的指标；下游任务基准（如 MMLU、HumanEval）评估量化对推理和代码能力的影响，更能反映实际应用中的性能；人工评估量化对对话质量的影响，捕捉自动化指标难以衡量的细微退化。在实际部署中，通常先在 Perplexity 上筛选候选方案，再在下游任务基准上验证，最后进行小规模的人工评估。
